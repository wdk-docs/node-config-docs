未来兼容性¶

为了确认更改相关软件所需的工作, node-config将会:

  • 使用 semver versioning 来促进稳定的部署,

  • 限制后向不兼容性以支持替代解决方案,

  • 为即将发布的不兼容版本提供 充足的警告

  • 尝试在不兼容的使用情况下检测并输出错误, 以及

  • 发布所有 即将发生的不兼容性 在此页面上

Semver版本控制¶

为了促进稳定部署, node-config在表达部署版本时使用 semver versioning.

这使得适合运行特定 MAJOR.MINOR 版本的测试, 并对指定通配符 PATCH 版本充满信心.例如, 如果您测试过的当前版本是 1.3.4, 那么 package.json 中的 node-config 依赖关系可能是:

"dependencies": {
  "config": ">=1.3.4 <1.4.0",
  ...

没有引入重大风险.您还可以指定 "config": "1.3.4", 如果您更喜欢版本稳定性来修复错误.

充足的警告¶

在不兼容性公告(下方)和包含不兼容性的版本的发布之间至少应给出一年的时间.

如果可以使用将来的不兼容性来检测您的应用程序, 则每个应用程序运行的一次性警告将被置于 console.error 中, 用于每种类型的不兼容性.

一旦引入了不兼容性, 如果检测到不兼容的用法, 则会在 console.error 中输入错误.

即将到来的不兼容性¶

版本2.0 - (不是在2015年7月之前)¶

  • 较少杂乱的顶级命名空间（2014年6月公布）实用程序函数 在 config.util 下可用`命名空间，因为版本1.0将不再在顶级 config 命名空间中可用。从版本2开始，它们只能在 config.util 下使用，并且 保留字 中的唯一单词list将是 get，has 和 util。

  • 不推荐使用Watch实用程序(2014年7月发布) watch 方法是作为已更改配置值的通知而构建的.从node-config 1.0开始, 不再建议配置更改, 并且监视功能保留在但不建议使用警告.手表功能在2.0版中不可用.
