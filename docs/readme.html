配置Node.js应用程序¶

      发行说明

介绍¶

Node-config为您的应用程序部署组织层次结构配置.

它允许您定义一组默认参数, 并为不同的部署环境(开发, qa, 登台, 生产等)扩展它们.

配置存储在应用程序中的 配置文件 中, 可以通过以下方式覆盖和扩展 环境变量, 命令行覆盖, 或者 从外部源配置.

这为您的应用程序提供了一个一致的配置界面, 该界面在 不断增长的npm模块列表中共享 也使用node-config.

项目指南¶

  • 简单 - 快速入门

  • 强大 - 用于多节点企业部署

  • 灵活 - 支持多种配置文件格式

  • 轻量级 - 小文件和内存占用

  • 可预测的 - 模块和应用程序开发人员的基础测试

快速开始¶

以下示例采用JSON格式，但配置可以在其他示例中 Configuration-Files-file-formats>.

安装在您的app目录中, 然后编辑默认配置文件.

$ npm install config
$ mkdir config
$ vi config/default.json

{
  // Customer module configs
  "Customer": {
    "dbConfig": {
      "host": "localhost",
      "port": 5984,
      "dbName": "customers"
    },
    "credit": {
      "initialLimit": 100,
      // Set low for development
      "initialDays": 1
    }
  }
}

编辑生产部署的配置覆盖:

$ vi config/production.json

{
  "Customer": {
    "dbConfig": {
      "host": "prod-db-server"
    },
    "credit": {
      "initialDays": 30
    }
  }
}

在代码中使用配置:

const config = require('config');
//...
const dbConfig = config.get('Customer.dbConfig');
db.connect(dbConfig, ...);

if (config.has('optionalFeature.detail')) {
  const detail = config.get('optionalFeature.detail');
  //...
}

config.get() 将为未定义的键抛出异常, 以帮助捕获拼写错误和缺失值.使用 config.has() 来测试是否定义了配置值.

启动您的应用服务器:

$ export NODE_ENV=production
$ node my-app.js

运行在这个配置中, dbConfig 的 port 和 dbName 元素将来自 default.json 文件, host 元素将来自 production.json 覆盖文件.
