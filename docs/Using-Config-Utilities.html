使用配置实用程序¶

Node-config附带了一组实现node-config所需的实用工具.这些实用程序在 config.util 对象中公开, 以便在您自己的应用程序中使用.

以下是这些实用程序的列表, 按一般用途顺序 - 您的里程可能会有所不同.

extendDeep(mergeInto, mergeFrom…, depth)¶

使用一个或多个对象(以及包含在其中的对象)扩展对象(及其包含的任何对象).

这并不像其他扩展函数那样替换深层对象, 而是潜入它们来扩展单个元素.如果多个扩展对象定义了相同的键, 则: * 如果所有相应的值都是对象, 则通过调用相同的函数递归合并它们. * 如果各个值都不是对象, 则尊重最新值. * 避免将对象和非对象混合为同一个键, 因为它可能导致意外行为.

param

type

描述

mergeInto

object

要合并的对象

mergeFrom…

object

要合并的任意数量的对象

depth

integer

可选的深度以防止递归.默认值:20.

(return)

object

返回更改的mergeInto对象

以下示例将两个对象的内容合并为一个新对象.

var newObject = config.util.extendDeep({}, baseObject, anotherObject);

cloneDeep(copyFrom, depth)¶

返回指定对象的深层副本.

这将返回一个新对象, 其中包含从指定对象复制的所有元素.深层副本由对象和数组组成, 因此您可以对返回的对象执行任何操作, 而不会影响输入对象.

param

type

描述

copyFrom

object

要从中复制的原始对象

depth

integer

可选的深度以防止递归.默认值:20.

(return)

object

具有从copyFrom对象复制的元素的新对象

例:

var copy = config.util.cloneDeep(fromObject);

equalsDeep(object1, object2, depth)¶

如果两个对象的内容相同, 则返回true.

param

type

描述

object1

object

要比较的对象

object2

object

要与之比较的对象

depth

integer

可选的深度以防止递归.默认值:20.

(return)

boolean

如果两个对象具有等效内容, 则为true

例:

var customerCopy = config.util.cloneDeep(myCustomer);
var same = config.util.equalsDeep(myCustomer, customerCopy); // <-- true
customerCopy.creditLimit = 7000;
var same = config.util.equalsDeep(myCustomer, customerCopy); // <-- false

diffDeep(object1, object2, depth)¶

返回包含两个对象之间不同的所有元素的对象.

当object2由深度复制object1发起时, 它最有效, 然后对object2进行了更改, 并且您需要一个对象, 该对象将为您提供对object1所做的更改, 从而导致object2.

param

type

描述

object1

object

要比较的对象

object2

object

要与之比较的对象

depth

integer

可选的深度以防止递归.默认值:20.

(return)

object

差异对象, 如果扩展到object1将导致object2.

makeImmutable(object, propertyName, propertyValue)¶

使javascript对象属性不可变(确保它不能从当前值更改).

如果未提供propertyName, 则该对象的所有属性都是不可变的.

属于对象的属性是递归调用的, 使子对象不可变.

可以将新属性添加到此(和子)对象, 并且除非在添加属性后调用此方法, 否则这些属性将不会是不可变的.

此操作无法撤消.

param

type

描述

object

object

包含要使其不可变的属性的对象

propertyName

string | [string]

(可选)要使其不可变的属性名称(或名称数组).如果未指定, 则对象的所有属性都是不可变的

propertyValue

mixed

(可选)在创建不可变之前将属性设置为的属性值.保留以实现向后兼容性, 仅在propertyName不是数组时使用.

(return)

object

原始对象, 具有新的不可变属性

例:

var a = {hello:'world'};
config.util.makeImmutable(a);
a.hello = 'there';
console.log ('Sorry, hello is still ' + a.hello);

makeHidden(object, propertyName, propertyValue)¶

使对象属性隐藏, 以便在枚举对象的元素时不显示.

该属性仍然存在, 可以读取和写入, 但它不会出现在 for ... in 循环, Object.keys() 或``JSON.stringify() ``类型方法.

param

type

描述

object

object

包含要隐藏的属性的对象

propertyName

串

要隐藏的属性的名称

propertyValue

串

(可选)将属性设置为此值

(return)

object

原始对象, 具有新隐藏的属性

例:

var a = {hello:"world'};
console.log ('Before hiding: ' + JSON.stringify(a));
config.util.makeHidden(a, 'hello');
console.log ('After hiding: ' + JSON.stringify(a));
console.log ('Hidden, but still there: ' + a.hello);

getEnv(varName)¶

获取配置环境变量的当前值

此方法返回指定的配置环境变量的值, 包括任何默认值或覆盖.

您可以检查的环境变量包括 NODE_ENV, CONFIG_DIR, NODE_CONFIG, HOSTNAME, NODE_APP_INSTANCE, 以及 环境变量 wiki页面.

param

type

描述

varName

串

环境变量名称

(return)

串

环境变量的值

例:

console.log('Configuration directory: ' + config.util.getEnv('CONFIG_DIR'));

loadFileConfigs(directory)¶

使用与主配置目录相同的约定读取给定目录(包括环境变量读取, 除了 NODE_CONFIG)并返回带有结果的对象.

如果没有给出目录, 则读取标准config目录并解析 NODE_CONFIG.

这对于与子模块一起使用很有用, 例如:

import config from 'config'
const ourConfigDir = path.join(__dirname, 'config')
const baseConfig = config.util.loadFileConfigs(ourConfigDir)
config.util.setModuleDefaults('MyModule', baseConfig)

toObject(config)¶

返回当前配置对象的新深层副本, 或者如果提供, 则返回配置的任何部分.

如果未提供 config (undefined), 则完整地转储当前配置对象.

param

type

描述

配置

object

要复制和序列化的配置部分.省略此参数以返回整个配置.

(return)

object

克隆的配置或配置的一部分

var result = config.util.toObject(config.get('db'));
