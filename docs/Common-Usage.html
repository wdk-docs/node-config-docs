常见用法¶

创建配置文件¶

创建你的 配置文件

加载节点配置¶

将它放在javascript文件的顶部:

var config = require('config');

或者加载特定的配置上下文:

var config = require('config').get('Customer');

使用配置值¶

访问配置的首选方法是通过 get():

var dbHost = config.get('dbConfig.host');

如果(当?)某人搞砸了你的配置文件, 这会给你一个快速失败的系统 - 它比你想象的更常见.在第一次调用 get() 时, 它也会使配置对象不可变.

如果配置文件不包含给定属性的值, get() 将抛出异常.注意 null 是一个 可接受的 值, 并且当 get() 返回它时不会抛出异常.如果传入get的参数是 null 或 undefined, Get也会抛出异常.

另外, 将配置值显式设置为 undefined 是一种有用的方法, 可以确保它们必须由以后的配置文件定义.

另一种方法是直接获得价值:

var dbHost = config.dbConfig.host;

这不能保护您免受拼写错误和配置文件问题或不可变性的影响.但它支持向后兼容性.

测试配置值¶

如果要查看配置值是否存在, 请使用 has() 方法

if (config.has('dbConfig')) {
  ...
}

这可以作为在 try/catch 块周围包装 get() 的替代方法. has() 不会抛出异常, 当且仅当配置文件具有所提供键的定义值时才会返回true. 注意 null 是一个定义的值.

has() 如果传递的参数是 null 或 undefined, 则不会抛出异常, 但只返回 false.
